# ADM 交付物指南

本指南详细描述 TOGAF ADM 各阶段的主要交付物和模板。

## Preliminary Phase 交付物

### 1. Architecture Principles（架构原则）

**用途**: 定义指导架构决策的基本原则

**模板结构**:
```
原则编号: [编号]
原则名称: [名称]
声明: [简短描述]
依据: [为什么需要这个原则]
影响: [原则对架构的具体影响]
```

### 2. Organizational Model for Enterprise Architecture

**用途**: 定义 EA 治理结构和组织

**内容要素**:
- EA 组织结构图
- 角色和职责定义
- 技能要求
- 报告关系

### 3. Tailored Architecture Framework

**用途**: 根据组织需求定制的架构框架

**内容要素**:
- 采用的方法和工具
- 定制的 ADM 流程
- 架构内容框架
- 术语表

## 最佳实践

1. **适度详细**: 根据项目规模和复杂度调整交付物详细程度
2. **保持一致**: 使用统一的模板和术语
3. **版本控制**: 对所有交付物进行版本管理
4. **定期更新**: 架构交付物需要随项目进展更新
5. **可追溯性**: 建立交付物之间的追溯关系
